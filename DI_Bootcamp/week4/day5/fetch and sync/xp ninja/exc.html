Promise Analysis Results

Check the console for the timed output. Below are the explanations for Exercises 2, 3, and 4.

Exercise 2: Promise.all

Outcome:
1. "==CONCURRENT START with Promise.all=="
2. "starting slow promise"
3. "starting fast promise"
4. (1s passes) "fast promise is done"
5. (Another 1s passes) "slow promise is done"
6. "slow"
7. "fast"
*Explanation: Promise.all starts both immediately. It waits for the LONGEST one (2s) 
before the .then() block runs and logs both results.*

Exercise 3: Parallel with await Promise.all

Outcome:
1. "==PARALLEL with await Promise.all=="
2. "starting slow promise"
3. "starting fast promise"
4. (1s passes) "fast promise is done" -> "fast"
5. (Another 1s passes) "slow promise is done" -> "slow"
*Explanation: Here, the inner async functions log their results AS SOON as they resolve. 
Unlike Ex 2, "fast" is logged before "slow" because the logging is inside the job.*

Exercise 4: Parallel with .then()

Outcome:
1. "==PARALLEL with Promise.then=="
2. "starting slow promise"
3. "starting fast promise"
4. (1s passes) "fast promise is done" -> "fast"
5. (Another 1s passes) "slow promise is done" -> "slow"
*Explanation: Standard parallel execution. Both start at once, and each logs its
